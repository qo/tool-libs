# Общие сведения

SIMD (single instruction, multiple data) - принцип компьютерных вычислений, позволяющий обеспечить параллелизм на уровне данных. Иными словами, SIMD - это описание компьютеров с несколькими процессорными компонентами, которые могут одновременно выполнять одну операцию с разными данными.

Большинство современных процессоров включают в себя короткие (64 или 128 бит) инструкции SIMD для повышения производительности для работы с мультимедиа. Так, на процессорах Intel используются такие SIMD-расширения как MMX, iwMMXt, SSE, SSE2, SSE3, SSSE3, SSE4.x, AVX и AVX2.

# Ход работы

В ходе работы требуется сравнить время исполнения двух программ:
- программы, которая вычисляет сумму чисел в массиве с помощью средств высокоуровневого языка программирования;
- программы, которая вычисляет сумму чисел в массиве с помощью SIMD-инструкций.

## Написание программы на высокоуровневом языке программирования

В качестве языка выбран язык программирования C. Ниже приведен код для вычисления суммы чисел в массиве.

```
#include <stdio.h>

int main() {

  double arr[] = {1.0, 2.0, 3.0, 4.0, 5.5};
  int size = sizeof(arr) / sizeof(arr[0]);
  
  double sum = 0;
  for (int i = 0; i < size; i++) {
    sum += arr[i];
  }

  printf("%f\n", sum);

  return 0;
}
```

Если скомпилировать данную программу, будет получен результат `15.500000`, что является верным.

Как можно видеть, данный процессор поддерживает такие SIMD-расширения, как SSE, SSE2, SSE3, SSSE3, SSE4.1, SSE4.2, AVX и AVX2.

На сайте Intel можно найти руководство [Intel Instrinsics Guide](https://www.intel.com/content/www/us/en/docs/intrinsics-guide/index.html), описывающее API, который можно использовать для вызова SIMD-инструкций. 

Было выбрано SIMD-расширение SSE3. Если в руководстве в качестве Instruction Set выбрать опцию SSE family - SSE2, можно получить список реализованных в данном расширении функций. Чтобы использовать SIMD-расширения SSE3 требуется подключить заголовочный файл `pmmintrin.h` (или любой более современный заголовочный файл для SIMD от Intel с обратной совместимостью с SSE3 - например `immintrin.h`). Также, в случае использования компилятора `gcc`, необходимо использовать флаг `-msseN`, где `N` - версия SSE.

Ниже приведен код для вычисления суммы чисел в массиве.

```
#include <stdio.h>

#include <immintrin.h>

int main() {

  double arr[] = {1.0, 2.0, 3.0, 4.0, 5.5};
  int size = sizeof(arr)/sizeof(arr[0]);

  __m128d vsum = _mm_set1_pd(0.0);

  for (int i = 0; i < size; i += 2) {
    __m128d v = _mm_load_pd(&arr[i]);
    vsum = _mm_add_pd(vsum, v);
  }

  vsum = _mm_hadd_pd(vsum, vsum);

  double sum = _mm_cvtsd_f64(vsum);

  printf("%f\n", sum);

  return 0;
}
```

Разберем все используемые здесь функции из заголовочного файла `immintrin.h`.

`_mm_set1_pd` используется для заполнения суммы элементов массива нулями.

`_mm_load_pd` используется для загрузки элементов массива в память. Следует отметить, что вообще говоря для использования этой функции элементы массива должны быть "выровнены". Функции для выравнивания объявлены в этом же заголовочном файле, однако для простоты выравнивание опускается. В связи с этим упрощением результат работы программы может быть не верен, однако в большинстве случаев такого не возникает.

`_mm_add_pd` используется для сложения текущей суммы с текущим элементом массива.

`_mm_hadd_pd` используется для сложения соседних пар значений типа `double` (64 бит) в сумме типа `__m128d` (128 бит) в "нижнюю" часть суммы.

`_mm_cvtsd_f64` используется для копирования "нижней" части суммы в финальный результат.

Если скомпилировать данную программу, будет получен результат `15.500000`, что является верным.

## Сравнение скорости

Код программ был изменен в строках, где объявлялся массив `arr`, с целью повышения его размера для более показательного сравнения времени работы непосредственно сложения.

Для сравнения скорости была использована утилита Linux `time`.

Для массивов размером 100000000 с помощью команд `time ./highlevel.out` и `time ./simd.out` были получены следующие замеры:

`./highlevel.out 3.76s user 0.35s system 99% cpu 4.118 total`

`./simd.out 2.66s user 0.32s system 99% cpu 2.991 total`

Данные замеры говорят о том, что с использованием SIMD-инструкций был получен прирост в 33% по производительности (4 секунды на исполнение в случае использования стандартных возможностей языка C, 3 секунды - в случае использования SIMD-инструкций).
